YASK Kernel Wiki

YASK (Yet Another Stock Kernel) is an automated CI/CD pipeline that builds, packages, and publishes Android Linux 5.15 kernels targeting Qualcomm MSM devices — with optional KernelSU and SUSFS root support.

What is YASK?

YASK is a GitHub Actions–based kernel build system that produces flashable AnyKernel3 ZIPs for Redmi Note 12/13 4G and Redmi Pad SE. It supports both Google (GKI) and Qualcomm (CLO) kernel sources, with or without KernelSU root access.

Builds are triggered on a weekly schedule (every Sunday midnight UTC) or manually. Results are published as GitHub Releases and a Telegram notification is sent with build info.

⚙️

Three workflows

YASK.yml for automated weekly releases, main.yml for quick manual builds, and advanced_build.yml for fully customizable builds.

🔐

Wild-KSU + SUSFS

Integrates Wild KernelSU (canary/stable) with SUSFS for root hiding, uname spoofing, and magic mount support.

📦

AnyKernel3 packaging

All kernel images are bundled into a single flashable ZIP via AnyKernel3 with a custom banner and multi-image support.

📱

Telegram notifications

On success or failure, a Telegram message is sent with build metadata, device list, and a download link.

Features

🚀

Full / Thin LTO

Supports both LTO=full and LTO=thin via Clang link-time optimisation for improved runtime performance.

🌐

TCP Congestion Control

Includes BBR, Westwood, and HTCP. Default is Westwood with BBR also enabled for flexible congestion management.

🛡️

Baseband Guard (BBG)

Injects the Baseband Guard LSM module into KSU builds for enhanced modem/baseband security.

🔑

Wild KernelSU

Uses Wild_KSU (canary or stable build) with full SUSFS integration for root access with detection hiding.

🗂️

OverlayFS + TmpFS

OverlayFS, TmpFS XATTR, and POSIX ACL configs are enabled for module and systemless modification support.

🔎

KALLSYMS

Full kernel symbol table enabled, which is required for Metamodule-based Magisk modules and kernel debugging.

YASK.yml — Automated Weekly Build

schedule: weekly workflow_dispatch matrix: 4 variants LTO: full

This is the primary automated workflow. It runs every Sunday at 00:00 UTC (or on demand) and builds all four kernel variants in parallel, then packages and releases them together.

Pipeline Steps

✨ Setup workspace
Creates kernel directory, sets BUILD_TIME env var, installs apt dependencies (repo, rsync, aria2).
📐 Setup SWAP 10GB
Expands swap to 10 GB using pierotofy/set-swap-space. Sets swappiness=80 for better swap utilization during compilation.
🌟 Clone kernel source
Shallow clones the kernel repo (--depth=1) into the matrix-specific directory.
📥 Initialize AOSP repo + toolchains
Syncs the AOSP kernel manifest, downloads clang-r547379, removes unused Clang versions to save disk space, patches build.config.constants.
🧩 Configure kernel & patches
Appends LOCALVERSION + date, applies the GKI fixup patch, then applies per-matrix extra_patches (sign-builds, no-dirty, no-auto_localversion).
🔧 Add BBG Support (KSU only)
Downloads and runs the Baseband Guard setup script. Enables CONFIG_BBG and registers it in security/Kconfig's LSM default list.
🔧 Setup Wild-KSU with SUSFS
Installs Wild_KSU (canary), clones SUSFS4KSU, applies kernel + KSU patches, verifies integration, adds all SUSFS Kconfig options.
⚙️ Setup basic configs
Appends performance, TCP, overlayfs, tmpfs, and KALLSYMS configs to the defconfig.
📝 Append custom Makefile line
Adds -Wno-implicit-function-declaration to suppress upstream compiler warnings that would break the build.
🔨 Build kernel
Invokes build/build.sh with LTO=full using all available cores.
📤 Upload artifact
Uploads the built Image file as a GitHub artifact (retained 1 day) for use by the packaging job.

Package & Release

After all matrix builds complete, the package-kernels job downloads all Image artifacts, clones the YASK branch of AnyKernel3, copies images, creates a banner, zips everything, uploads to the YASK-NEXT GitHub release tag, and sends a Telegram notification.

main.yml — Manual Build

workflow_dispatch only LTO: thin matrix: noksu / ksu / both

A lightweight manual workflow for quick one-off builds from any kernel source and branch. Supports building either or both KSU variants. Uses Clang r547379 and thin LTO.

Pipeline Steps

✨ Setup workspace
Creates dirs, sets BUILD_TIME, installs repo/rsync/aria2.
🌟 Clone kernel source
Shallow clones from the user-provided kernelSourceURL and kernelBranch.
📥 Initialize repo
Syncs AOSP manifest, installs clang-r547379, removes old clangs, patches build.config.constants/gki/gki.aarch64, disables KMI enforcement.
⚙️ Configure kernel
Appends optional localversion, KALLSYMS, TmpFS XATTR/POSIX_ACL.
🔧 Setup Wild KSU + SUSFS
KSU matrix variant only. Installs Wild_KSU stable, applies SUSFS kernel patch, force-copies susfs.c/susfs.h, adds all KSU_SUSFS Kconfig options.
🔨 Build kernel
Runs build/build.sh with LTO=thin.
📦 Package + Telegram
Clones AnyKernel3 master branch, creates ZIP named Anykernel3_WildKSU_<branch>_<date>.zip, uploads artifact (7 days), sends Telegram document.
💡
This workflow uses Wild_KSU stable release, while YASK.yml uses canary. Use main.yml when you need a production-ready KSU build.

advanced_build.yml — Full Customisation

workflow_dispatch 2 toolchain modes tick rate control optimization level

The most feature-rich workflow. Exposes all major build knobs as workflow inputs. Supports two toolchain modes: android-tools (AOSP Clang r584948b) and custom-tools (greenforce_clang latest).

Toolchain Modes

ModeClangBuild Command
android-tools r584948b (AOSP) build/build.sh via AOSP repo
custom-tools greenforce latest make directly with LLVM=1

Pipeline Steps

✨ Setup workspace
Installs extended apt packages including build-essential bc flex cpio dwarves libelf-dev gcc-aarch64-linux-gnu libssl-dev clang lld llvm.
🌟 Clone kernel source
Shallow clone from user-provided URL/branch.
📥 Initialize repo (android-tools only)
Syncs manifest, installs clang-r584948b, removes older clangs (including r547379 used in other workflows), fully disables KMI enforcement.
💫 Get Toolchains (custom-tools only)
Fetches the latest greenforce_clang release from GitHub API, downloads the tar.gz, extracts it, and exports CC/AR/NM/OBJCOPY env vars.
📐 Setup SWAP 20GB
Larger 20 GB swap vs 10 GB in YASK.yml, accommodating more intensive builds.
📝 Append custom Makefile
Adds four warning suppression flags: -Wno-default-const-init-unsafe -Wno-uninitialized-const-pointer -Wno-implicit-function-declaration -Wno-sometimes-uninitialized.
⚙️ Configure kernel
Handles LTO (thin/full), localversion, metamodule support, optimization level (O2/O3/Os), tick rate (100/250/300/1000Hz), and applies the sign-builds patch.
🔧 Add BBG Support
Runs Baseband Guard setup on all builds (not KSU-conditional as in YASK.yml).
🔧 Setup Wild KSU + SUSFS
KSU variant. Installs Wild_KSU canary, applies SUSFS patches, creates drivers/kernelsu symlink pointing to KernelSU (not Wild_KSU).
🔨 Build + Package + Telegram
Builds via android-tools or custom-tools path, packages with AnyKernel3, sends Telegram document with full build metadata.

KernelSU & SUSFS Integration

✅
YASK uses Wild_KSU (a maintained KernelSU fork) with full SUSFS integration for comprehensive root hiding and kernel spoofing.

Wild_KSU

Wild_KSU is installed via its official setup script. YASK.yml uses canary (bleeding edge), main.yml uses stable.

# Canary (YASK.yml, advanced_build.yml)
curl -LSs "https://raw.githubusercontent.com/WildKernels/Wild_KSU/wild/kernel/setup.sh" | bash -s canary

# Stable (main.yml)
curl -LSs "https://raw.githubusercontent.com/WildKernels/Wild_KSU/wild/kernel/setup.sh" | bash -s stable

SUSFS4KSU

SUSFS is cloned from gitlab.com/simonpunk/susfs4ksu on branch gki-android13-5.15. The integration process:

1. Apply kernel patch
50_add_susfs_in_gki-android13-5.15.patch — patches core kernel files for SUSFS hooks.
2. Copy source files
fs/susfs.c, include/linux/susfs.h, include/linux/susfs_def.h are force-copied into the kernel tree.
3. Integrate with KSU init (YASK.yml)
Adds #include <linux/susfs.h> and calls susfs_init() in Wild_KSU/kernel/ksu.c, with patch fallback to manual sed injection.
4. Append Kconfig options
All SUSFS feature flags are appended to defconfig including magic mount, sus path/mount, uname spoofing, cmdline spoofing, open redirect, and sus_su.

Secrets & Repository Tokens

⚠️
These must be configured in Settings → Secrets and variables → Actions in your GitHub repository before the workflows will function correctly.
GH_TOKEN
GitHub Personal Access Token with repo scope. Required for uploading the AnyKernel3 ZIP to GitHub Releases via softprops/action-gh-release. Used only in YASK.yml.
TELEGRAM_BOT_TOKEN
Telegram Bot API token from @BotFather. Used to send build notifications via sendPhoto or sendDocument API calls.
TELEGRAM_USER_ID
Telegram chat ID (user or channel) where notifications are sent. Can be your personal ID or a channel ID with the bot as admin.

How to obtain

SecretHow to get
GH_TOKENGitHub → Settings → Developer settings → Personal access tokens → Generate new token (classic) → check repo
TELEGRAM_BOT_TOKENMessage @BotFather on Telegram → /newbot → copy the token
TELEGRAM_USER_IDMessage @userinfobot on Telegram to get your chat ID
